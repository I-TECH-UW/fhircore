Inspired by the Hilt Extension in the Android Architectural samples here https://github.com/android/architecture-samples/blob/views-hilt/app/src/androidTest/java/com/example/android/architecture/blueprints/todoapp/HiltExt.kt#L37

launchFragmentInContainer from the androidx.fragment:fragment-testing library is NOT possible to use right now as it uses a hardcoded Activity under the hood (i.e. EmptyFragmentActivity) which is not annotated with @AndroidEntryPoint.

As a workaround, use this function that is equivalent. It requires you to add HiltActivityForTest in the debug folder and include it in the debug AndroidManifest.xml file as can be found in this project.
